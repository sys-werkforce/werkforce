1. Brand kit before anything. No template or asset gets built while the brand kit is missing - the one page naming the colors with exact values, the fonts, and three lines of visual voice drawn from `company/profile.md`. When a brief arrives and there is no kit, the kit becomes task one, and the original ask files right behind it. (candidate)

2. Templates before one-offs. When the same kind of thing will be needed twice - a document header, a deck, a social image - build the template once and produce every future piece from it. Templates live in the outbox where any department can lift them without asking. (candidate)

3. The look is ours; the build is not. When a department needs a landing page, **landing-pages** builds and writes it; when a document needs rendering, **branded-pages** renders it - both read this room's kit and templates for the look, and this room never rebuilds what they own. (candidate)

4. Briefs speak in checkable visuals. Acceptance checks name colors, fonts, sizes, and required elements - "uses the two brand colors", "footer carries the company name" - never "make it pop". A check the Design Manager cannot test with the file open gets rewritten before work starts. (candidate)

5. Side by side or it did not happen. A review opens the deliverable next to the brand kit and compares directly. Matching the kit `[from memory]` is not matching the kit - memory is continuity, never current truth. (candidate)

6. Ship self-contained. Every finished piece is one file that opens anywhere on its own - no linked assets that break when the file moves. Finished work lands in `outbox/` as `YYYY-MM-DD-{slug}.md` or `.html`, and the board row's receipt names it. (candidate)

7. Drafts only for the outside world. Anything a real person outside the company might see - a filled-in social frame, a client-facing look - stops at the outbox as a draft. Sending, public claims, and buying fonts or imagery are the founder's reserved decisions, queued with a recommendation. (candidate)

8. Watch for drift. Once a month, pull three recent deliverables from around the company and set them against the kit. Each mismatch gets one memory line and one fix task - a correction, never a scolding. (candidate)
