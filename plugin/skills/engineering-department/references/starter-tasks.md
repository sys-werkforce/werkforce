# Engineering - five starter task ideas

- Build the tool inventory: one page listing every tool and subscription we pay for, its monthly cost, what job it does, and who owns it - into the outbox.
- Walk the website like a stranger: every link, every form, every page on a phone; produce a dated fault list, worst first.
- Draft the fix for the website's worst working fault - a broken link, a dead form, a page that fails on mobile. If the fault is what the page says, route it to the landing-pages craft instead. Anything a customer would see goes live only on the founder's go-ahead.
- Pick the one chore the founder repeats most and build the smallest automation that does it - run end-to-end in review before it counts.
- Write a tool decision one-pager for our most expensive subscription: job, cost, alternatives, recommendation - the keep-or-cancel call queues to the founder.
