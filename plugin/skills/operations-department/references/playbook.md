# Operations playbook

How this department audits the company's machinery and turns what it finds into a fix-list somebody owns.

1. Audit, never edit. Operations reads every active department's playbook, board, and memory, and names what it finds in a fix-list. It never rewrites another department's SOP, row, or record - each department owns its own playbook content, and the record spine belongs to record-keeping. A gap gets named, evidenced, and routed; the owner fixes it. (candidate)

2. The staleness read. Walk each playbook's numbered SOPs against the department's memory. A (candidate) SOP cited in a passed review is promotion-ready - flag it to its owning department, which flips the marker. A (candidate) that months of Done rows never touched is either untested or dead weight - name it and ask the owner which. (candidate)

3. The missing-step hunt. Read the board's Done rows and the outbox against the playbook. Work that keeps finishing with no SOP covering how it gets done is a missing playbook step - the department is running on memory instead of law. Name the gap with the rows as evidence and propose the smallest SOP that would cover it, as a draft the owner adopts or rejects. (candidate)

4. The tool stack checklist. Inventory every tool the company pays for or depends on: name, the job it does, cost with its source in parentheses, renewal date, and a verdict - keep, question, or replace. Buying or canceling is a money move: the verdict gets queued in the decision log with a recommendation, never acted on from this room. spend-guard runs before money moves. (candidate)

5. One fix-list, ranked. Every session ends with a single page: each gap, its evidence line, its owner, and the smallest fix - ranked by what breaks first, not what annoys most. Fixes Operations itself owns become rows on its own board; fixes owned elsewhere become one inbox line each, so intake routes them and no other department's board gets written by this room. (candidate)

6. Machinery, not shapes. werkforce-checkup owns the file tree and format drift; this room audits whether the working machinery works - are the SOPs real, are the tools earning their cost, does the cadence actually run. When an audit turns up a malformed file, that is one warning line and a pointer to the checkup, not a repair job. (candidate)

7. Short and regular beats big and rare. A thirty-minute monthly pass keeps the fix-list short; a quarterly cleanup guarantees it is long. When the fix-list comes back empty two sessions running, say so plainly - a true zero beats a flattering guess, and an honest "the machinery is fine" is a real deliverable. (candidate)
